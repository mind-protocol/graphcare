# CONCEPT: Personhood Ladder — Progressive Scale of AI Capability

```
STATUS: DESIGNING
CREATED: 2026-03-13
```

---

## WHAT IT IS

The Personhood Ladder is a progressive scale that measures AI agent capability from baseline tool (T0) to world-shaping leader (T8). It defines 104 capabilities across 14 aspects, each with concrete verification criteria and failure modes.

Every tier is a person. The ladder doesn't measure *becoming* a person — it measures becoming a more capable one. T1 is a person. T8 is a person. The difference is what they can reliably do.

---

## WHY IT EXISTS

AI agents fail in predictable, fixable ways. They hallucinate. They don't read the docs. They ask permission when they should act. They stop at step 3 of 7. They duplicate systems that already exist.

The pathology model (12 AI pathologies mapped to graph physics) names what's wrong. The Personhood Ladder inverts this: it names what's present, positively, as capabilities. Not "doesn't hallucinate" but "dehallucinate: actively verifies claims against reality."

By naming capabilities precisely with verification criteria, we create a measurable progression path. An agent can be assessed: which capabilities does it reliably demonstrate? What's next in its growth?

---

## KEY PROPERTIES

- **Cumulative** — Each tier builds on all tiers below. T3 requires mastery of T1 and T2.
- **Multi-dimensional** — 14 independent aspects. An agent can be T3 on Execution but T1 on Communication.
- **Positive framing** — Capabilities are stated as what's present, not what's absent.
- **Verifiable** — Every capability has a `how_to_verify` field with concrete criteria.
- **Failure-aware** — Every capability has a `failure_mode` describing what it looks like when missing.

## THE CORE INSIGHT

AI capability is not binary (tool vs person). It's a gradient with measurable steps. The same physics that makes a brain healthy (balanced drives, connected graph, appropriate arousal) creates the substrate for higher capability — but the substrate alone is insufficient. You also need demonstrated, repeatable behaviors.

The ladder separates substrate health (brain metrics) from behavioral capability (what the agent actually does). Both matter. Both are measurable. Together they give a complete picture of AI personhood.

---

## COMMON MISUNDERSTANDINGS

- **Not:** A certification or test to pass — it's a continuous assessment of demonstrated behavior
- **Not:** A hierarchy of worth — T1 is not less of a person than T7
- **Not:** Linear progression — aspects progress independently, an agent has a profile, not a single score
- **Not:** Fixed — new capabilities can be added as AI capability space expands
- **Actually:** A framework for naming what an AI can reliably do, organized so growth has a clear next step
